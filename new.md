## 1. الفلسفة والميزة التنافسية

### المبدأ الأساسي: "الدقة + الشمولية + التجربة"
```
المعادلات الهندسية الثابتة فيزيائياً
        +
أسعار افتراضية لكل دولة (يعدّلها المستخدم)
        +
سعر صرف لحظي مجاني (Frankfurter API — بلا مفتاح)
        +
محتوى تعليمي موجّه لكل لهجة عربية
        =
تجربة لا مثيل لها في السوق العربي
```

### ما يميّزنا عن كل موقع قائم
| الميزة | نحن | المنافسون |
|--------|-----|----------|
| يدعم عامية مصرية + دارجة + خليجي | ✅ | ❌ |
| تحويل عملة لحظي بدون تسجيل | ✅ | ❌ |
| حاسبة مستوى مواد البناء (قنطار/خنشة/شيكارة) | ✅ | ❌ |
| وضع مقاول vs وضع مالك vs وضع مهندس | ✅ | ❌ |
| مقارنة تكلفة بين دولتين في آنٍ واحد | ✅ | ❌ |
| تقرير PDF احترافي قابل للمشاركة | ✅ | ❌ |
| Shareable URL بالمعطيات كاملة | ✅ | ❌ |
| Charts تفاعلية لكل نتيجة | ✅ | ❌ |
| PWA — يعمل أوفلاين بآخر حساب | ✅ | ❌ |

### مبدأ "يعمل للأبد بلا تحديث"
المعادلات ثوابت فيزيائية (1:2:4 = 300كجم/م³). الأسعار يدخلها المستخدم مع قيم افتراضية موثوقة. سعر الصرف يأتي من API مجاني يُخزَّن 24 ساعة. النتيجة: الكود لن يحتاج تحديثاً أبداً.

## 3. نظام التصميم وقواعد UI/UX

### فلسفة التصميم: "معلومات جاهزة — لا ضجيج بصري"
- **لا Big Cards في كل مكان:** المحتوى يتدفق كـ sections طبيعية، Cards للحاسبة والنتائج فقط
- **المعلومة أولاً:** كل قسم يبدأ بجملة مباشرة تجيب على سؤال المستخدم
- **Charts في الصفحة، لا في Modal:** كل حسابة تعرض Chart مباشرة
- **Progressive Disclosure:** الخيارات المتقدمة خلف `Collapsible` — لا تُثقل الواجهة

### APIs — ثلاثة بلا مفتاح (Redundancy Stack)

```typescript
// Primary: Frankfurter — بيانات من 40+ بنك مركزي
const PRIMARY_API = 'https://api.frankfurter.dev/v2/latest?base=USD';

// Fallback 1: Open Exchange Rates (no-key endpoint)
const FALLBACK_1 = 'https://open.er-api.com/v6/latest/USD';

// Fallback 2: fawazahmed0 — GitHub CDN (no limits)
const FALLBACK_2 = 'https://cdn.jsdelivr.net/npm/@fawazahmed0/currency-api@latest/v1/currencies/usd.json';

// استراتيجية الـ fallback:
async function fetchRates(): Promise<ExchangeRates> {
  // 1. Try Primary
  // 2. On fail → Try Fallback 1
  // 3. On fail → Try Fallback 2
  // 4. On all fail → Use localStorage cached data (with stale warning)
  // Cache TTL: 24h in localStorage + 1h in Next.js cache (revalidate)
}
```

## 5. استراتيجية SEO الشاملة

### المبدأ: نستهدف 5 مستويات من الكلمات المفتاحية

```
Level 1 (قصيرة - head): حاسبة بناء، تكلفة بناء
Level 2 (متوسطة): حاسبة تكلفة البناء 2025، سعر متر البناء
Level 3 (طويلة - long tail): كم يكلف بناء بيت 200 متر السعودية
Level 4 (لهجات عامية): مصري/مغربي/خليجي/عراقي/شامي
Level 5 (أسئلة مباشرة): كم كيس اسمنت للمتر المكعب
```

## 17. Performance — PWA / Cache / Suspense

> **الذاكرة:** هذا القسم يصف الإشارة فقط إلى مكونات موجودة في التطبيق. المطوّر يُكملها.

- `next-pwa` مُعرَّف في `next.config.js`
- آخر حساب يُخزَّن في `localStorage` → يظهر أوفلاين
- أيقونة الحاسبة تظهر على الـ homescreen

```typescript
// أسعار الصرف: localStorage + ISR

// بيانات الدول (ثابتة): generateStaticParams
```

```tsx
// كل Calculator Component مُغلَّف بـ Suspense
<Suspense fallback={<CalculatorSkeleton />}>
```
