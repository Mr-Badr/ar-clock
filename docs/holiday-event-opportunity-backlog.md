# Holiday Event Opportunity Backlog

Research date: 2026-06-15

Use this backlog with [`docs/holiday-event-premium-content-playbook.md`](./holiday-event-premium-content-playbook.md).

When you pick an event from this backlog and want another AI to generate the actual event files, give it [`docs/add-new-event.md`](./add-new-event.md) plus the event name. That file is the canonical AI handoff contract for creating only the correct source files, with the required SEO, indexing, data, Arabic language, and validation conditions.

The rule for this backlog is stricter than a normal SEO ideas list: include only events with a stable, specific date that we can keep correct without waiting for a yearly announcement. The goal is to publish rich Arabic event pages that are useful to users, crawlable for Google, and safe for Google Ads review.

## Stable-Date Rule

An event can stay in this backlog only if it has one of these date patterns:

- fixed Gregorian date, such as `20 November`
- official annual observance with the same Gregorian date every year
- country-specific fixed public holiday with a reliable official or institutional source

Do not add events here if the date depends on:

- moon sighting or official religious announcement
- school calendars that change every academic year
- daylight-saving or clock-change decisions that can be changed by government announcement
- campaign themes that must be rewritten every year to remain accurate
- one-off holidays or bridge holidays declared for a single year

If an event has a fixed date but the paid day off can be moved by government decision, the page can still be created. The event date must stay fixed, and the content must clearly separate "event date" from "official vacation date".

## Scoring Model

| Score | Meaning |
|---|---|
| Demand | Likely Arabic search volume during the event window |
| Gap | How weak or fragmented Arabic results look |
| Fit | How naturally Miqatona can win with date, countdown, country, and planning value |
| Difficulty | Estimated top-3 difficulty for a premium Arabic page |

Difficulty is not a guarantee. "Easy" means the intent is narrow, the Arabic SERP is likely thin, or existing results are mostly generic calendars/news.

These may still be valuable later, but they are not allowed in this fixed-date backlog because they can become wrong without active yearly updates.

## Content Angle That Should Beat Competitors

For every selected event, use this page promise:

```text
أعطيك الجواب السريع أولاً: التاريخ، اليوم، العد التنازلي، وهل هي إجازة أم مناسبة فقط.
ثم أشرح لك الشيء الذي تربكك فيه معظم النتائج: هل التاريخ ثابت؟ هل العطلة قد تنتقل؟ ما الفرق بين هذه المناسبة ومناسبة قريبة منها؟ وماذا يمكنك أن تفعل بهذا التاريخ في المدرسة أو العمل أو السفر أو المحتوى؟
```

This is where Miqatona can win. Many Arabic pages answer "what is it" but fail to answer "what do I do with this date?"

## Validation Before Creating Each Event

Before building any item from this list, run the premium playbook:

```text
Follow docs/holiday-event-premium-content-playbook.md.
Use top 5 Arabic and top 5 English results.
Scrape and read the competitor pages for structure, gaps, missing questions, and user intent.
Do not copy or lightly paraphrase competitor text.
Verify official sources before writing any factual claim.
Create only package.json, research.json, qa.json.
Run events:build, validate:holidays:slug, seo:validate, test:unit, and build.
```

For fixed events whose public vacation may be moved, keep `dateRule` fixed and explain the vacation caveat in visible content.
