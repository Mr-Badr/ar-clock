# دليل إضافة المحتوى الغني — Content Author Guide

> هذا الملف يشرح كيف تضيف محتوى نصياً طويلاً لأي مناسبة دون كسر الكود.  
> اقرأه بالكامل قبل أي تعديل.

---

## 1. أين تكتب المحتوى؟

المحتوى الغني موجود **فقط** داخل مجلد:

```
src/lib/event-content/
├── islamic.js          ← المناسبات الإسلامية العامة (رمضان، عيد، حج...)
├── seasonal.js         ← المناسبات الفصلية (رأس السنة، بداية الصيف...)
└── countries/
    ├── sa.js           ← المحتوى الخاص بالسعودية
    ├── eg.js           ← مصر
    ├── dz.js           ← الجزائر
    ├── ma.js           ← المغرب
    ├── ae.js           ← الإمارات
    ├── tn.js           ← تونس
    ├── kw.js           ← الكويت
    └── qa.js           ← قطر
```

**القاعدة الذهبية:**
- مناسبة إسلامية عامة (رمضان، عيد الأضحى) → `islamic.js`
- مناسبة خاصة بدولة (اليوم الوطني السعودي) → `countries/sa.js`
- مناسبة فصلية (إجازة الصيف) → `seasonal.js`

المفتاح في كل الملفات هو **الـ slug** — مثل `'ramadan'` أو `'saudi-national-day'`.

## 2. بنية المحتوى الكاملة (Full Schema)

```js
'your-event-slug': {

  // ══════════════════════════════════
  // SEO — مطلوب
  // ══════════════════════════════════

  seoTitle: 'متى رمضان {{year}} / {{hijriYear}} — عد تنازلي',
  // العنوان الذي يظهر في Google. يجب أن يحتوي السنة الديناميكية {{year}}.
  // الحد الأقصى: 60 حرفاً.

  description: 'رمضان {{year}} يبدأ ... كم باقي على رمضان؟',
  // وصف الصفحة لـ Google. الحد الأقصى: 155 حرفاً.

  keywords: [
    'كم باقي على رمضان {{year}}',
    'موعد رمضان {{hijriYear}}',
    'رمضان {{nextYear}} موعد',
    // أضف كلمات مفتاحية طويلة long-tail — ستُضاف تلقائياً لكلمات SEO
  ],

  // ══════════════════════════════════
  // المحتوى النصي الغني — اختياري لكن مهم للـ SEO
  // ══════════════════════════════════

  about: {
    heading: 'عن شهر رمضان المبارك',
    // العنوان يظهر فوق الفقرات

    paragraphs: [
      'فقرة أولى — 60 إلى 100 كلمة. اذكر اسم المناسبة والسنة {{year}} في أول سطر.',
      'فقرة ثانية — تاريخ أو خلفية المناسبة. يمكنك كتابة فقرات طويلة هنا بلا حدود.',
      'فقرة ثالثة — التفاصيل العلمية أو الدينية أو الاجتماعية.',
      // يمكنك إضافة عدد غير محدود من الفقرات!
    ],
  },

  significance: 'نص أثري أو حديث نبوي أو آية قرآنية — يظهر في بوكس مميز.',
  // اختياري. يظهر كاقتباس بارز في الصفحة.

  // ══════════════════════════════════
  // حقائق سريعة — تظهر كجدول
  // ══════════════════════════════════

  quickFacts: [
    // نوع ثابت (القيمة لا تتغير):
    { label: 'عدد أيام الصيام', value: '29 أو 30 يوماً' },

    // نوع ديناميكي — التاريخ الميلادي يُحسب تلقائياً:
    { label: 'التاريخ الميلادي', _dynamic: 'gregorian' },
    // ⚠️ لا تكتب value — النظام يحسبه من targetDate

    // نوع ديناميكي — التاريخ الهجري يُحسب تلقائياً:
    { label: 'التاريخ الهجري', _dynamic: 'hijri' },
    // ⚠️ يعمل فقط مع المناسبات من نوع hijri

    // نوع مع token في القيمة:
    { label: 'موعد {{year}}', value: '18 فبراير {{year}}' },
    // {{year}} سيُستبدل بالسنة الصحيحة تلقائياً
  ],

  // ══════════════════════════════════
  // العادات والتقاليد — تظهر كبطاقات
  // ══════════════════════════════════

  traditions: [
    { icon: '🌙', title: 'الصيام', description: 'الامتناع عن الطعام من الفجر حتى المغرب.' },
    { icon: '📖', title: 'تلاوة القرآن', description: 'يحرص المسلمون على...' },
    // أضف عدداً غير محدود من البطاقات
  ],

  // ══════════════════════════════════
  // الجدول الزمني — اختياري
  // ══════════════════════════════════

  timeline: [
    { phase: 'أول رمضان', description: 'بداية الصيام وصلاة التراويح.' },
    { phase: 'العشر الأواخر', description: 'أفضل أيام رمضان.' },
  ],

  // ══════════════════════════════════
  // مواعيد الدول — يظهر كجدول مقارنة
  // ══════════════════════════════════

  countryDates: [
    { country: 'السعودية', flag: '🇸🇦', code: 'sa', date: '18 فبراير {{year}}', note: 'أم القرى' },
    { country: 'مصر',      flag: '🇪🇬', code: 'eg', date: '18–19 فبراير {{year}}', note: 'دار الإفتاء' },
    // ⚠️ استخدم {{year}} وليس 2026 المحددة
  ],

  // ══════════════════════════════════
  // الأسئلة الشائعة — مهمة جداً للـ SEO
  // ══════════════════════════════════

  faqItems: [
    {
      q: 'متى يبدأ رمضان {{year}}؟',
      // السؤال يجب أن يطابق البحث الفعلي على Google بالضبط

      // الإجابة يجب أن تكون مباشرة ومفيدة — لا تقل "يُحسب تلقائياً"

    {
      q: 'متى رمضان {{nextYear}}؟',
      // 🔑 أضف سؤالاً عن السنة القادمة لالتقاط حركة البحث المبكر
      a: 'رمضان {{nextYear}} سيبدأ حوالي 7 فبراير — يتقدم ~11 يوماً كل عام.',
    },
    // يمكنك إضافة 5-10 أسئلة أو أكثر
  ],

  // ══════════════════════════════════
  // مقالات ذات صلة — روابط داخلية للـ SEO
  // ══════════════════════════════════

  relatedSlugs: ['eid-al-fitr', 'laylat-al-qadr', 'ramadan-in-saudi'],
  // ⚠️ يجب أن تكون slugs موجودة في src/lib/events/

  // ══════════════════════════════════
  // المصادر والمراجع — اختياري
  // ══════════════════════════════════

  sources: [
    { label: 'تقويم أم القرى', url: 'https://www.ummulqura.org.sa' },
  ],
},
```

## 3. نظام الـ Tokens الديناميكية

هذا النظام يضمن أن كل تاريخ دائماً صحيح بدون تعديل:

| Token | يُستبدل بـ | مثال |
|---|---|---|
| `{{year}}` | السنة الميلادية الحالية | `2026` → `2027` تلقائياً |
| `{{nextYear}}` | السنة القادمة | `2027` → `2028` تلقائياً |
| `{{hijriYear}}` | السنة الهجرية الحالية | `1447` → `1448` تلقائياً |

**🔴 قاعدة حرجة:** لا تكتب أبداً:
```js
// ❌ خطأ — هذا سيصبح قديماً بعد ديسمبر 2026
seoTitle: 'متى رمضان 2026',
value: '18 فبراير 2026',

// ✅ صحيح — يتحدث تلقائياً
seoTitle: 'متى رمضان {{year}}',
value: '18 فبراير {{year}}',
```

### `_dynamic` في quickFacts

استخدم `_dynamic` بدلاً من `value` للبيانات التي تُحسب من تاريخ المناسبة:

```js
// ❌ خطأ — ستعطي نفس الواجهة لعدة سنوات
{ label: 'التاريخ الميلادي', value: '18 فبراير 2026' }

// ✅ صحيح — يحسب التاريخ الصحيح من قاعدة AlAdhan API
{ label: 'التاريخ الميلادي', _dynamic: 'gregorian' }
{ label: 'التاريخ الهجري',   _dynamic: 'hijri' }
```

> ⚠️ `_dynamic: 'hijri'` يعمل فقط مع المناسبات من نوع `type: 'hijri'` في ملفات `src/lib/events/`.

## 4. الحقول المحظورة في المحتوى

هذه الحقول **مملوكة لملفات الأحداث** ولا تُكتب في ملفات المحتوى:

```js
// ❌ لا تضعها في event-content/ — ستُحذف تلقائياً من النظام
category, _countryCode, type, slug, id,
hijriMonth, hijriDay, month, day, date, __enriched
```

هذا الحماية موجودة في `event-content/index.js` في دالة `getRichContent()`.

## 5. خطوات إضافة محتوى لمناسبة موجودة

**مثال:** إضافة محتوى غني لمناسبة `eid-al-adha` الموجودة:

1. **افتح الملف الصحيح:** `src/lib/event-content/islamic.js`
2. **ابحث عن المفتاح:** `'eid-al-adha': {`
3. **أضف أو عدّل الحقول** بحسب البنية أعلاه
4. **تحقق من عدم وجود أرقام ثابتة** مثل `2026` — استخدم `{{year}}`

## 6. خطوات إضافة محتوى لمناسبة جديدة

> **قبل إضافة محتوى، يجب أن تكون المناسبة معرّفة كحدث في `src/lib/events/`.**

### الخطوة 1: تعريف الحدث في `src/lib/events/`

```js
// مثال في src/lib/events/islamic.js
{
  id:        'taraweeh',          // معرف فريد
  slug:      'taraweeh',         // يُستخدم في الـ URL /holidays/taraweeh
  name:      'صلاة التراويح',     // الاسم العربي
  type:      'hijri',            // hijri | fixed | estimated | monthly | easter
  hijriMonth: 9,                 // شهر رمضان
  hijriDay:   1,
  category:  'islamic',          // الفئة — من القائمة المسموحة أدناه
},
```

**الفئات المسموحة فقط:**
```
islamic | national | school | holidays | astronomy | business | support
```

### الخطوة 2: إضافة المحتوى في الملف الصحيح

```js
// في src/lib/event-content/islamic.js — أضف بعد آخر مناسبة:
'taraweeh': {
  seoTitle: 'صلاة التراويح {{year}} — متى تبدأ وكم ركعة',
  description: '...',
  about: {
    heading: 'عن صلاة التراويح',
    paragraphs: ['...', '...'],
  },
  faqItems: [...],
},
```

### الخطوة 3: التحقق

```bash
npm run build
```

إذا انتهى البناء بدون أخطاء — المحتوى يعمل بشكل صحيح.

## 7. أنواع المناسبات وكيف تؤثر على المحتوى

| النوع | معنى | مثال |
|---|---|---|
| `hijri` | تاريخ هجري — يُحسب من AlAdhan API | رمضان، العيد، الحج |
| `fixed` | تاريخ ثابت كل عام | اليوم الوطني (23 سبتمبر) |
| `estimated` | تاريخ تقريبي قابل للتغيير | بداية الدراسة |
| `monthly` | يتكرر كل شهر | موعد الراتب |
| `easter` | مرتبط بعيد القيامة | شم النسيم |

> مع نوع `hijri`: يمكنك استخدام `_dynamic: 'hijri'` و `_dynamic: 'gregorian'` في quickFacts لأن النظام يعرف التاريخين.

## 8. قواعد الكتابة للـ SEO العربي

1. **أول فقرة** يجب أن تحتوي اسم المناسبة + `{{year}}` + كلمة مفتاحية رئيسية
2. **أسئلة FAQ** يجب أن تطابق الاستعلامات الفعلية على Google — لا تكتب أسئلة لن يبحث عنها أحد
3. **لا تكرر** نفس الكلمة مرتين في نفس السؤال
4. **أضف سؤالاً** عن `{{nextYear}}` لالتقاط حركة البحث المبكر
5. **الفقرات** — 60 إلى 150 كلمة لكل فقرة — يمكنك كتابة 5-10 فقرات أو أكثر
6. **countryDates** — استخدم `{{year}}` في الـ date دائماً

## 10. ماذا تفعل إذا وجدت مناسبة بدون محتوى؟

المناسبة ستعمل عادياً — ستُعرض مع محتوى افتراضي مُولَّد من اسمها. لكن إذا أردت محتوى غنياً:

1. ابحث عن الـ slug في `src/lib/events/` لتعرف نوع المناسبة ودولتها
2. افتح الملف الصحيح في `src/lib/event-content/`
3. أضف المفتاح والمحتوى باتباع البنية أعلاه
4. شغّل `npm run build` للتحقق

لا حاجة لتعديل أي ملف آخر.
